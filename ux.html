Project Name

Scotiabank Mobile App Update: A Human Centered UX Case Study

Problem Statement

This project simulates the kind of UX research and redesign work a Product Designer or UX Researcher might conduct for a digital banking team.

Using dummy survey responses and fictional user personas, I analyzed patterns in how different user groups—students, gig workers, and full-time employees—engage with Scotiabank’s savings goal feature.

The case study explores how human-centered design methods such as:

  • Survey analysis
  • Journey mapping
  • Persona creation
can uncover usability barriers and inform improved user flows.

Key techniques included:

  • Synthesizing survey data
  • Identifying behavioral trends
  • Prototyping enhanced app features using Figma

Objectives

To improve the savings goal experience for Scotiabank users, this project set out to accomplish the following objectives:

  • Identify core user pain points through stakeholder interviews and journey mapping
  • Visualize the current user flow and highlight friction points using Miro
  • Prototype interface improvements in Figma to address navigation and clarity issues
  • Present a redesigned user flow and interface to guide future development decisions

Methodology

To guide the redesign, I applied a Design Thinking framework supported by quantitative and qualitative research tools

Define

To clarify these simulated user needs, I created and distributed a dummy survey designed to resemble what a real banking UX team might deploy. The responses (120+ fictional entries) were crafted to reflect varied user profiles across occupations and age groups. Questions focused on confidence during goal setup, clarity of features like "round-ups" and "funding rules," and whether the app offered personalized savings guidance.

View the full survey here.

A pattern emerged: students and part-time workers consistently reported confusion around language and lack of feedback. Spreadsheet analysis confirmed that users who selected "Not confident" also frequently chose "Understanding the options" as the hardest part of the process. These insights informed the creation of two persona's Adam and Fatima. Both their goals and pain points shaped the rest of the design process.

Ideate

Using insights from the fictional persona journeys and dummy survey responses, I simulated a sketching sprint to explore potential improvements. I prioritized concepts that addressed clarity, language, and flow, organizing ideas into key themes: navigation, visual guidance, and confirmation feedback. These simulated brainstorming exercises reflect the kind of early ideation a UX team might conduct when redesigning a feature.

Prototype

I translated the top ideas into mid-fidelity wireframes using Figma . These included a progress bar to anchor users in the flow, simplified terminology (e.g., “auto-save” instead of “funding rule”), and a visual goal summary at the end of the setup. An interactive prototype allowed for real-time demonstration of the improved experience.

Test & Iterate

I shared the prototype with peers for usability feedback. Revisions included stronger button hierarchy and step-by-step guidance for auto-save options. Additional microcopy was added to reduce uncertainty.

Present

I shared the simulated prototype with peers acting as stand-in users to gather usability feedback. While not a formal user test, this peer review simulated the kind of feedback a UX team might collect during early-stage testing. Based on the input, I revised the design to include a clearer button hierarchy, step-by-step guidance for auto-save options, and added microcopy to reduce user uncertainty.
