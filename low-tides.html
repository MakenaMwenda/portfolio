Project Name

Simulated Amazon Rollout: Excel VBA Automation for Readiness Tracking

Project Description

As part of a simulated Amazon project management team, I supported the rollout of a new internal tech tool across global logistics and office teams. The tool aims to improve scheduling, communication, and task tracking. My responsibility was to monitor rollout readiness, automate tracking workflows using Excel macros, and generate status reports to inform leadership on progress and risks.

Objective

  • Create a composite Readiness Score to quantify rollout preparedness across teams.
  • Automate project workflows using Excel Macros and VBA for efficiency and repeatability.
  • Generate dynamic, filterable reports to support leadership decision-making.
  • Ensure the workbook supports weekly updates, visual tracking, and future macro expansion.

Methodology

Data Preparation

I cleaned the dataset by converting binary fields (e.g., Infra Ready, Training Complete, Checklist Complete) from "Y/N" to 1/0 using helper columns. This standardization was necessary for quantitative scoring and further automation.

Readiness Score Calculation

I created a composite Readiness Score (0–100) based on five weighted inputs that reflect overall team preparedness:

  • Infrastructure Ready – 25%
  • Training Completion % – 25%
  • Checklist Complete – 20%
  • Issue Backlog Score – 15%
  • Stakeholder Confidence – 15%

The formula penalizes incomplete training, unresolved infrastructure, and high backlog scores, while rewarding strong stakeholder confidence.

VBA Macro Development

I wrote a macro named StatusReportGenerator that:

  • Prompts the user to filter by Region or PM Owner
  • Copies the filtered results into a new worksheet titled Status Report
  • Preserves formatting and optionally resets filters on the original data
  • Is triggered via an interactive button placed on the main worksheet

This macro is structured to be modular and easily extendable for additional filter types or formatting rules.

Interactive Elements

I added a form control button on the Excel sheet that allows one-click execution of the macro. This enables non-technical users to generate tailored reports without navigating the VBA editor. The interface design prioritizes ease of use and supports integration into a broader dashboard.

Results

  • The dashboard identified engagement peaks at 11 AM, 2 PM, and 6 PM, with longer dwell times on weekends.
  • “Wing Left” and “Entrance” zones had the highest interaction frequency.
  • A Python script was used to automate alerts when dwell time KPIs dropped below defined thresholds.
  • Community survey data confirmed strong support from Mount Pleasant and Kitsilano residents and showed a demand for interactive and co-created installation types.
